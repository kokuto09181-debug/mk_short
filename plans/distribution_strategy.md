# 拡散戦略プラン

## 概要

「知られざる偉人伝」チャンネルの視聴者獲得・拡散戦略。  
ターゲットは**50代以上の歴史好き**。YouTube Shorts を起点に、複数プラットフォームへ展開してどこに顧客がいるかを把握する。

---

## コンテンツ構造

```
長編動画（10分・別パイプライン）
    └── 冒頭・見どころを切り抜き
         ↓
Shorts / リール（45〜60秒）  ← 現在自動生成中
         ↓
各プラットフォームへ同時投稿
         ↓
反応を観測 → 顧客がいる媒体に集中投資
```

---

## 投稿プラットフォーム一覧

### 優先度 高

| プラットフォーム | 形式 | 理由 | 実装方法 |
|----------------|------|------|---------|
| **YouTube Shorts** | 動画 | 現行稼働中 | 自動（既存） |
| **X (Twitter)** | 動画クリップ + テキスト | 歴史好きコミュニティが活発。拡散力が高い | Twitter API v2 |
| **note** | テキスト記事 + 動画埋め込み | 歴史好きの読書層。SEO効果あり。脚本をほぼ流用可能 | note API |

### 優先度 中

| プラットフォーム | 形式 | 理由 | 実装方法 |
|----------------|------|------|---------|
| **Instagram リール** | 動画 | 50代女性リーチが厚い | Meta Graph API |
| **Facebook リール** | 動画 | 50〜60代が最も活発なSNS | Meta Graph API |

### 優先度 低（将来検討）

| プラットフォーム | 形式 | 備考 |
|----------------|------|------|
| Yahoo!クリエイターズ | 動画 / 記事 | 審査あり。高齢層への到達力は高い |
| SmartNews | 記事 | 申請制。通過すれば大リーチ |

## 各プラットフォームの投稿内容

### X (Twitter)
- 動画（60秒以内）または サムネイル画像 + YouTube リンク
- テキスト: タイトル + 「続きはYouTubeで」+ ハッシュタグ
- ハッシュタグ例: `#偉人 #日本史 #歴史雑学 #知られざる歴史 #shorts`
- 投稿タイミング: 動画公開と同時（07:00 / 12:30 / 20:00）

### note
- 脚本をそのまま記事として投稿（加筆不要）
- 末尾に「動画版はこちら → YouTube リンク」
- 画像: Wikipedia 画像 or Pexels 画像を流用
- タグ: `偉人` `日本史` `歴史` `歴史解説` `雑学`

### Instagram / Facebook リール
- 同一動画ファイルを投稿
- キャプション: タイトル + ハッシュタグ
- 日英チャンネルで別アカウント運用

## 市場調査としての活用

複数媒体に投稿し、**2〜4週間後に以下の指標を比較**する。

| 指標 | 確認項目 |
|------|---------|
| インプレッション | どの媒体で最も表示されたか |
| エンゲージメント率 | いいね・コメント・保存率 |
| 流入元 | YouTubeのトラフィックソース分析 |
| フォロワー増加速度 | どの媒体でファン化が起きているか |

→ 反応が出た媒体にリソースを集中する

## 実装ロードマップ

### Phase 1 — X への自動投稿（工数: 小）
- [ ] Twitter API v2 のキー取得
- [ ] `src/x_poster.py` 作成（動画 or サムネイル + テキスト投稿）
- [ ] `pipeline.py` の YouTube アップロード後に呼び出し追加
- [ ] GitHub Secrets に `TWITTER_*` キーを追加

### Phase 2 — note への自動投稿（工数: 小）
- [ ] note API（非公式）または手動投稿フローの確立
- [ ] `src/note_poster.py` 作成（脚本 → Markdown 変換 → 投稿）
- [ ] 記事テンプレートの作成

### Phase 3 — Instagram / Facebook リール（工数: 中）
- [ ] Meta for Developers でアプリ申請
- [ ] `src/meta_poster.py` 作成
- [ ] 日英別アカウントの設定

### Phase 4 — 効果測定・集中（工数: 継続）
- [ ] 各媒体のアナリティクスを週次で確認
- [ ] 反応の薄い媒体は停止、反応のある媒体を強化
- [ ] Notion に「媒体別パフォーマンス」列を追加して管理
